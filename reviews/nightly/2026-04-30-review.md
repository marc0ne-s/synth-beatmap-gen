## Summary
A quick-hitting change that adds a learnable layer-norm residual gate for coarse-memory fusion in the beatmap transformer. The math is clean and the idea is sound — stabilising the coarse-memory injection with normalisation is a standard and sensible move. A few sharp edges around parameter grouping and the results bookkeeping need attention, but nothing that'll blow up training tonight.

## Actionable Items

- **Fragile parameter filtering.** The optimizer splits parameters by string-matching names (`"coarse_proj" not in n`, `"fusion_ln" not in n`). Rename a module later and the LR grouping silently breaks. Use direct module references or `id(p)` sets instead of name regexing.
- **LayerNorm init on residual addition.** `x = self.fusion_ln(x + coarse_feat)` norms the already-activated sum of audio, target, difficulty *and* coarse embeddings. If `x` has been through a non-linearity or positional encoding with large variance, the norm will re-scale everything — not just the coarse residual. Verify this doesn't dampen the audio/difficulty signal unexpectedly. Consider norming only the coarse branch (`x + self.fusion_ln(coarse_feat)`) or applying norm pre-fusion if you want strict residual behaviour.
- **Results.tsv drift.** The new row (`3110fcb`) is being added in the same diff as the code change, but `3110fcb` is the *prior* commit (it added the PRD). The score row likely corresponds to the old `coarse_proj` fusion, not the new `fusion_ln` version. Make sure baseline labels are accurate — tag it `baseline no-ln` or similar so you don't benchmark against a moving target.
- **Parameter grouping side-effect.** `coarse_proj_params` was already getting `max_lr`. Now `fusion_ln` joins it, which is fine, but `LayerNorm` weight/bias don't always benefit from aggressive LR schedules. If you see instability, split `fusion_ln` to `base_lr` and keep only the projection on `max_lr`.
- **Minor:** `list(model.coarse_proj.parameters())` is unnecessary — `parameters()` returns an iterator that's concat-able directly. Harmless, but noisy.
