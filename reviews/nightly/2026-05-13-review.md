# Nightly Code Review: SynthGen Project
**Date:** 2026-05-13
**Scope:** `/Volumes/Second-Brain-1/AI/Synth`

## Summary
The latest changes represent a significant pivot from a Tauri-based desktop application to a web-focused editor. The project structure has been cleaned up, removing Tauri-specific dependencies and refactoring the `WelcomeScreen` to handle `.synth` file loading via JSZip. The core logic for loading maps is now integrated into the UI flow.

## Issues & Observations

### 1. Logic & Architecture
- **File Loading Fragility:** In `WelcomeScreen.tsx`, the `.synth` file validation (checking for ZIP magic numbers) is a good start, but the metadata extraction logic is highly dependent on specific file structures (`synthriderz.meta.json` vs `beatmap.meta.bin`). If the schema changes, the `setProject` call will fail or produce invalid state.
- **State Management:** The `setProject` call in `tryLoadSynth` initializes the project state. Ensure that `useProject` handles the transition from a "null" project to a loaded one cleanly, especially regarding cleanup of previous project data.
- **Dependency Management:** The removal of `@tauri-apps/*` packages suggests a move to a pure web environment. Ensure that any local file system access previously handled by Tauri is properly accounted for (e.g., via File System Access API if targeting modern browsers).

### 2. Code Quality & Idioms
- **Type Safety:** In `WelcomeScreen.tsx`, `raw` is typed as `any`. While acceptable for a quick prototype, this should be replaced with a proper interface once the `.synth` metadata schema is finalized to prevent runtime errors during `setProject`.
- **Error Handling:** The `try...catch` in `tryLoadSynth` uses `alert()`. While functional for an MVP, this should be transitioned to a more robust UI notification system (e.g., a toast component) to avoid blocking the main thread and providing a better user experience.
- **Refactoring Opportunity:** The `ToolButton` and `SnapButton` components in `EditorScreen.tsx` were refactored to use single-line arrow functions. This is fine for brevity but can impact readability if the logic grows.

### 3. Security
- **Zip Bomb Vulnerability:** Using `JSZip` to load user-provided files without size limits or depth checks could expose the browser to "zip bomb" attacks. Since this is a local/desktop-focused tool, the risk is lower but still present.
- **Path Traversal:** While not explicitly visible in the diff, ensure that when extracting files from the `.synth` archive, no files are being written to arbitrary paths on the user's system.

### 4. Performance
- **Lazy Loading:** The use of `await import('jszip')` is a good practice to keep the initial bundle size small.
- **Re-renders:** In `EditorScreen.tsx`, the `useHotkeys` dependencies are correctly handled, but ensure that `setTool` and `setSnap` calls don't trigger unnecessary heavy re-renders of the entire `Timeline` component.
